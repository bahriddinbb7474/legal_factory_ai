# UI Concept

The first web interface should be practical and work-focused.

## Layout

- Left side: chat history and new chat button.
- Center: large chat window.
- Bottom center: message input and file upload control.
- Above input: agent selection and risk/escalation status.
- Right side: document editor for drafts and generated legal documents.
- Top area: token and dollar cost indicator.
- Bottom left: user profile, role, and settings.

## Main Screens

- Login.
- Chat workspace.
- Document editor panel.
- Settings for profile, roles, agents, models, and costs.
- Approval view for red-risk items.

## Design Direction

The interface should feel like an internal legal operations tool: clear, dense enough for daily work, and focused on scanning risk, sources, and next actions.
