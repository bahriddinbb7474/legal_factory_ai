# Stages

## 1. Project Foundation

Create repository structure, documentation, environment template, and future module folders.

## 2. Web UI Skeleton

Build login, role-aware layout, left chat history, central chat area, bottom input, user profile, and settings entry points.

## 3. Chat and Agents

Add basic chat creation, message storage, and placeholders for three legal agents.

## 4. OpenRouter Integration

Connect OpenRouter with configurable models, disabled-by-default provider behavior, usage tracking, and fake providers for tests.

## 5. File Upload and Document Analysis

Support PDF, DOCX, XLSX, TXT, and images. Store files locally and bind them to chats.

## 6. Right Document Editor

Add a right-side document editor for letters, claims, government replies, legal opinions, disagreement protocols, HR orders, and memos.

## 7. RAG and Legal Sources

Index internal documents and trusted legal sources. Require citations for legal conclusions.

## 8. Roles and Approval

Implement roles, document statuses, red-risk approval, and approval comments.

## 9. Cost Tracking

Track input tokens, output tokens, model usage, and cost by chat and agent.

## 10. Telegram Bot

Add Telegram question entry, document upload, risk notifications, web links, and approve/reject actions.

## 11. Testing and Launch

Verify the web UI, chat, agents, file upload, document editor, RAG, approval, cost tracking, and audit journal.
