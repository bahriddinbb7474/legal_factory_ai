# Documentation map

Legal Factory AI documentation is grouped by purpose:

- [`00_current/`](00_current/) — read first: active project context, specification/roadmap, architecture, decisions, tests, and risks.
- [`10_policies/`](10_policies/) — normative prompt, RAG, response, verdict/document, section-routing, and legal-source lifecycle policies.
- [`20_sources/`](20_sources/) — legal source overview, approved registry, and admin maintenance instructions.
- [`30_guides/`](30_guides/) — development, roles, UI, and AI-agent guides.
- [`40_stage_reports/`](40_stage_reports/) — historical implementation reports and acceptance evidence grouped by stage.

Read in this order:

1. [`00_current/CONTEXT.md`](00_current/CONTEXT.md) — project passport and current snapshot.
2. [`00_current/SPEC_AND_ROADMAP.md`](00_current/SPEC_AND_ROADMAP.md) — product scope, roadmap, and priorities.
3. [`00_current/TESTS_AND_RISKS.md`](00_current/TESTS_AND_RISKS.md) — verification baseline, required tests, risks, and safety limits.
4. [`00_current/ARCHITECTURE.md`](00_current/ARCHITECTURE.md) — technical architecture and current/legacy boundaries.
5. [`00_current/DECISIONS_LOG.md`](00_current/DECISIONS_LOG.md) — accepted and pending decisions.

Then read only the policy, source, guide, or historical report relevant to the active task. Files under `40_stage_reports/` are evidence, not current project truth.
